## 1. Цели документа

1. Любой пост по понедельникам, средам, пятницам, и т.д. — пишется по фиксированной структуре. Никакой импровизации каждое утро.
2. Автоматический генератор `daily_post_generator.py` получает один JSON-промпт на день недели → выдаёт черновик в нужном формате.
3. Любой подрядчик / agent / LLM получает регламент и работает в одном tone of voice без отдельного инструктажа.
4. FACT-CHECK PROTOCOL встроен в каждый шаблон — нельзя сгенерировать пост без явных источников.

## 2. Общие принципы (применимо ко всем 7 рубрикам)

### 2.1 Tone of voice — словами

| Делаем | Не делаем |
|---|---|
| Простой язык. Сложные термины — с расшифровкой при первом упоминании в посте | Сыпать аббревиатурами без пояснений |
| Конкретные цифры, объекты, улицы, имена компаний | «Один интересный объект», «известный застройщик» |
| Спокойная уверенность, короткие предложения | Восклицательные знаки, эмоциональные крюки |
| Личный взгляд — «я думаю», «по моему опыту», «мне это не нравится потому что» | «Доверьтесь профессионалу», «индивидуальный подход» |
| Признание сомнений, открытое мнение с маркером | «Гарантирую», «100%», «точно» |

### 2.2 Языковой стоп-лист (твёрдый)

- ❌ «Премиум», «элитный», «эксклюзив», «эксклюзивно»
- ❌ «Успей», «только сегодня», «осталось N мест»
- ❌ Гарантия доходности в %
- ❌ «Юридически чист на 100%» → правильно: «проверяем по ФЗ-214 совместно с юристами»
- ❌ «Инсайдер» в публичной коммуникации → правильно: «эксперт изнутри рынка»
- ❌ «Рекомендую», «индивидуальный подход», «доверьтесь профессионалу»
- ❌ AI-аватары, голос-клоны, сгенерированные фото объектов
- ❌ Голословные анти-обзоры ЖК / БЦ / застройщиков

### 2.3 Аббревиатуры — что расшифровывать

**Расшифровывать при первом упоминании в посте:**
- NOI = чистый операционный доход (аренда минус расходы и налоги)
- IRR = внутренняя норма доходности
- Cap rate = коэффициент капитализации (NOI / стоимость покупки)
- ДКПБН = договор купли-продажи будущей недвижимости
- ДДУ = договор долевого участия
- УК = управляющая компания
- ПВ = первоначальный взнос
- ЦБ = Центральный банк
- CPI = индекс потребительских цен (показатель инфляции)
- HR = подбор и удержание персонала

**Можно использовать без расшифровки** (общеизвестно для нашей ЦА):
- БЦ, ЖК, ТЦ, ТРЦ
- Москва-Сити, Сити (применительно к ММДЦ)
- ТТК, МКАД, Садовое кольцо
- ИП, ООО, ЮЛ

### 2.4 Длина постов — точные диапазоны

| Тип | Знаков с пробелами | Время чтения |
|---|---|---|
| Цифра дня / опрос | 400–700 | 30–45 сек |
| Дайджест недели | 800–1 200 | 60–90 сек |
| Разбор / сравнение / методология | 1 200–2 200 | 90–150 сек |
| Голосовое (сценарий) | 1 200–1 700 | 60–90 сек устной речи |
| Лонгрид-актив (раз в 2 нед) | 2 500–3 500 | 3–4 мин |

**Правила:** длиннее 3 500 — режем на 2 поста. Короче 400 — сводим в «цифру дня».

### 2.5 Универсальная структура поста

```
[ХУК — 1 строка, 80–120 знаков]
Конкретика, цифра, парадокс, прямой вопрос. Без вводок «Сегодня поговорим о…».

[КОНТЕКСТ — 2–3 строки]
Почему это важно прямо сейчас. Кому это касается.

[ТЕЛО — 2–4 смысловых блока]
Списком, подзаголовками или связным текстом. Каждый блок ≤ 3 строк.
Цифры. Объекты. Имена. Источники.

[ВЫВОД — 1–2 строки]
К чему пришли. Без морали и пафоса.

[CTA — опционально, только в одном посте из недели]
Если есть — мягкий, на одно действие.

[ИСТОЧНИКИ — обязательно для постов с цифрами]
📨 Источник: @channel, дата, прямая ссылка
```

### 2.6 FACT-CHECK маркеры (по протоколу `.business/_rules/FACT-CHECK-PROTOCOL.md`)

Каждая цифра в посте должна иметь явный тип источника:

| Тип | Как помечать в посте | Когда применимо |
|---|---|---|
| **Verified** | прямая ссылка на источник | публичная статистика, торги, новости |
| **Опыт Ивана** | «по моему опыту», «в моих 12 годах работы», «у моих клиентов» | методологии, наблюдения |
| **Расчёт** | «считаю на цифрах», «по моим прикидкам» | NOI, IRR, сравнения |
| **Открытое мнение** | «я думаю», «моё мнение», «мне кажется» | прогнозы, оценки |

**Запрещено:** цифры без явного типа источника. Если непонятно — не публикуем.

### 3.1 Понедельник — «Разбор объекта недели»

**Цель:** дать читателю готовый разбор конкретного БЦ / ритейл-лота / коммерческого помещения, который сейчас на рынке. Главный продукт канала по будням.

**Сегмент:** коммерция-инвест + офис под бизнес (двух сегментов сразу).

| Параметр | Значение |
|---|---|
| Длина | 1 200–2 200 знаков |
| Картинка | ✅ обязательна (шаблон «metrics-card») |
| Голос | — |
| CTA | — (раз в месяц — мягкий: «хочешь разбор твоего варианта — в личку») |

**Структура (фиксированная):**
1. Хук с одним фактом-цифрой об объекте (1 строка)
2. Контекст: какой объект, когда вышел, чем особенный (3–4 строки)
3. **Что получает покупатель** — список 3–5 пунктов с цифрами
4. **Кому это подходит** — список 2–4 типов покупателя
5. **Кому НЕ подходит** — список 1–2 типов
6. Вывод: твоё мнение в одном абзаце
7. Источник по объекту: ссылка на оригинальный пост застройщика / СМИ

**Источники фактуры (откуда брать объект недели):**
1. `market-intel/digest/topics/kommerciya-bc.md` — приоритет «свежее + важность 3+»
2. Telegram-каналы: @propertyinsider, @stone_developer, @mrgroupoffice, @novostroyman, @SamoletPlusAgents, @osnovabrokers, @nedvizha
3. CRE.ru, Cushman & Wakefield, NF Group — отчёты

**Чек-лист fact-check для разбора:**
- [ ] Цена за м² — со ссылкой на пост застройщика или СМИ
- [ ] Условия рассрочки / ипотеки — со ссылкой
- [ ] Параметры объекта (этажи, метраж, дата ввода) — со ссылкой
- [ ] Любая оценка ликвидности — с маркером «открытое мнение» или «расчёт»
- [ ] Сравнение с другими объектами — с маркером «опыт Ивана»

**Картинка — шаблон `card-review.html`:**
- Заголовок: название объекта (PT Serif, 76px, акцентное слово терракотовым)
- Подзаголовок: локация · метраж · ключевая характеристика
- Блок «metrics» — 3 строки: цена, этажи/метраж, условия
- Футер: бренд + handle

### 3.3 Среда — «Сравнение / расчёт на бумаге»

**Цель:** наглядно показать математику двух альтернатив. Самый сильный формат для конверсии в личку (без прямого CTA).

**Сегмент:** инвесторы (жильё + коммерция) + офис под бизнес.

| Параметр | Значение |
|---|---|
| Длина | 1 200–1 800 знаков |
| Картинка | ✅ обязательна (шаблон «compare-card», 2 колонки) |
| Голос | — |
| CTA | — |

**Структура:**
1. Хук — типичное возражение клиента в кавычках (1 строка). Пример: «Положу 200 млн на депозит под 18% — заработаю больше, чем на офисе»
2. Контекст: разберём на цифрах (1 строка)
3. **Вариант 1** — название + 4–5 строк расчёта
4. **Вариант 2** — название + 4–5 строк расчёта
5. **Что меняет картину** — 1–3 фактора, которые не видны в лоб
6. Вывод: твой взгляд, почему сравнение в лоб неверно
7. Источники: ставки / аренда / цены — ссылки

**Темы для среды (банк на лето):**
- Депозит vs офис класса А (5 лет)
- Купить vs арендовать офис (10 лет, конкретная компания)
- Жилой инвест vs коммерческий лот (по доходности и горизонту)
- БЦ Сити vs БЦ Белорусская (по NOI и ликвидности)
- Стрит-ритейл vs офис класса А (по управлению)
- Один большой лот vs три мелких (по диверсификации)
- Депозит vs ОФЗ vs коммерция (тройное сравнение)
- Москва Запад vs Москва Восток (по росту цен)

**Чек-лист fact-check:**
- [ ] Все цифры по аренде / доходности / ставкам — со ссылками или маркером «расчёт»
- [ ] Прогноз ставки ЦБ / роста цен — с маркером «открытое мнение» + источник прогноза
- [ ] NOI — обязательно с указанием формулы

**Картинка — шаблон `card-compare.html`:**
- Заголовок с цифрой и горизонтом
- 2 колонки: левая (нейтральная рамка) — депозитный вариант, правая (терракотовая рамка) — коммерческая альтернатива
- В каждой: 2–3 строки расчёта + итого
- Внизу — фраза-выровненный вывод

### 3.4 Четверг — «За кулисами недели» (голосовое)

**Цель:** живой формат, личность Ивана. Через голос — высший фактор доверия в TG.

**Сегмент:** все.

| Параметр | Значение |
|---|---|
| Длина сценария | 1 200–1 700 знаков |
| Длина устной речи | 60–90 секунд в спокойном темпе |
| Картинка | — |
| Голос | ✅ обязательно |
| CTA | — |

**Структура сценария:**
1. Зачин — что произошло на этой неделе (1 фраза для «настройки»)
2. Конкретное наблюдение / инсайт (4–6 строк)
3. Что значит / почему важно (3–4 строки)
4. Вывод-правило для слушателя (2–3 строки)
5. Короткий финал — без морали и без CTA

**Темы для голоса (банк на лето):**
- Что спрашиваю на показе, чего не спрашивает покупатель
- Почему я отказываюсь от 1 сделки из 5
- Что увидел в работе УК БЦ — поразило
- Реакция на свежую новость (Moscow Towers, ставка ЦБ, новый закон)
- Личная история: один кризис из 4 (2014 / 2020 / 2022 / 2024)
- 2 случая, когда клиент мне не поверил — и что вышло через год
- Что я понял за 12 лет, чего не было в начале
- Разговор с финдиректором клиента — три вопроса, на которые он не смог ответить
- Что я ненавижу в работе брокера — и как с этим живу

**Правила подачи (для Ивана):**
- Тон — как разговор с одним человеком, не как «выступление»
- Не торопиться. Паузы — норма, они работают
- Произнести вступление дважды, выбрать живую версию
- НЕ читать с листа дословно — лучше прочесть 2 раза, отложить телефон и сказать своими словами

**Что НЕ работает в голосе:**
- Сложные таблицы, графики, расчёты с цифрами (в голосе плохо запоминаются)
- Длинные списки пунктов (теряется внимание)
- Юридические формулировки

**Чек-лист fact-check:**
- [ ] Любая цифра в голосе — verified (читатель не сможет проверить, доверие выше — ответственность выше)
- [ ] Если рассказ про конкретный объект — без раскрытия NDA, обобщённо
- [ ] Опираться на «опыт Ивана», избегать «открытое мнение»

### 3.5 Пятница — «Методология / Red flags» + CTA

**Цель:** дать практичный чек-лист, который сохраняют. Единственный пост недели с прямым CTA в личку.

**Сегмент:** все (через коммерческий фокус).

| Параметр | Значение |
|---|---|
| Длина | 1 500–2 200 знаков |
| Картинка | — (по умолчанию). По желанию — карточка-обложка |
| Голос | — |
| CTA | ✅ обязательно, мягкий |

**Структура:**
1. Хук с обещанием конкретного количества пунктов (1 строка). Пример: «7 вещей, которые я лично проверяю в коммерческом помещении до сделки»
2. Контекст: когда применяется чек-лист (1–2 строки)
3. **N пунктов** (5–10), каждый — 2–3 строки
4. Вывод: что важно понимать в принципе
5. **CTA:** мягкое приглашение применить чек-лист к их варианту

**Темы для методологии (банк на лето):**
- 7 red flags при покупке коммерческого помещения
- 5 вопросов арендатору ДО подписания договора
- Как сравнить 2 БЦ класса А — методология
- Что проверить в управляющей компании до покупки лота
- Юридический чек-лист по ДКПБН на коммерцию
- Как считать реальный NOI (с расходами, которые забывают)
- 6 ошибок при выборе офиса под собственный бизнес
- Как читать договор аренды от лица собственника
- 5 вопросов брокеру застройщика, на которые у него редко есть ответ

**Шаблоны CTA (мягкие, выбирается один на пост):**

## 6. FACT-CHECK PROTOCOL — операционный режим

**Перед каждой публикацией:**

1. **Прогон через `real-estate-expert`** — он сверяет каждую цифру и факт с правилами из `.business/_rules/FACT-CHECK-PROTOCOL.md`
2. **Любая цифра без источника** → пост помечается красным флагом «🔴 ТРЕБУЕТ ПРОВЕРКИ» при доставке в @IGDeveloper_bot
3. **Иван видит результат проверки в боте** перед публикацией:
   - 🟢 — всё проверено, можно публиковать
   - 🟡 — есть «открытые мнения» (это нормально, но Иван должен видеть)
   - 🔴 — есть цифры без источника → не публиковать, переделать

**Запрещено:**
- LLM-выдуманные цифры рынка
- Цифры из памяти LLM (ставки, проценты вакансии и т.п.)
- Прогнозы без указания автора прогноза (ВТБ / Сбер / CBRE / Cushman)

## 7. Метрики качества контента (отслеживать еженедельно)

| Метрика | Откуда | Триггер вмешательства |
|---|---|---|
| ER (открытие, реакции, переходы) | TG-аналитика канала | < 8% к 4-й неделе — переделать тон / темы |

| «Да активно» в опросе → отвечено в личке | CRM + ассистент | < 30% доходимости — пересмотреть скрипт первого сообщения |
| Доля постов с маркером «🔴 fact-check» | от автомата | > 10% — переписать промпт-шаблон рубрики |
| Доля постов, опубликованных без правок Иваном | от автомата | < 60% — переделать шаблон рубрики |
| Лиды → встреча (квалифицированный разговор) | CRM | < 2/нед к августу — пересмотреть CTA-шаблоны |

## 8. Что делать, если рубрика «не пишется»

**Если в market-intel нет свежей фактуры по нужному дню:**

| День | Запасной вариант |
|---|---|
| Пн (разбор) | Возьми объект, который ты лично знаешь (без свежей публикации). Маркер «опыт Ивана». |
| Вт (цифра дня) | WebSearch по 1 запросу за последние 30 дней. Или цитата прогноза ВТБ / Сбер / ЦМАКП. |
| Ср (сравнение) | Тема из банка — на цифрах прошлого квартала с пометкой «по данным [месяц]». |
| Чт (голос) | Реакция на новость с прошлой недели + личное правило. |
| Пт (методология) | Твой опыт — не зависит от свежих новостей. Темы из банка. |
| Сб (опрос) | Темы из банка. |
| Вс (дайджест) | Если событий мало — делай 3–4 события вместо 5–7, длина 600–800 знаков. |

**Если случается перерыв 1 день:**
- Никаких извинений в канале
- Сразу следующий день — по плану

**Если перерыв 2+ дня:**
- Один пост-«ремарка» с обновлением (что готовится, без подробностей)
- Дальше — по плану
